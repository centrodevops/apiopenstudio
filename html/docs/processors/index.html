The following can be used in the processor section of a resource.

Concatenate

Concatenate a series of strings or numbers into a single value.

  • Inputs
    • input
      An array of 2 or more values. Accepts processor or literal.

Example:

processor: concatenate
meta:
  id: 1
  input:
    - val1
    - processor: varStr
      meta:
        id: 2
        value: middlestr
    - val2

Field

Create a key: value pair. This is primarily for use in Object processor.

  • Inputs
    • Key
      The key of the NVP. Accepts processor or literal.
    • Value
      The value of the NVP. Accepts processor or literal.

Example:

processor: field
meta:
  id: 1
  key: key1
  value: val1

Filter

Filter values out of a data-set.

  • Inputs
    • values
      the data-set to filter
    • filter
      the values to filter out

Example:

processor: filter
meta:
  id: 1
  values:
    - val1
    - val2
    - processor: varStr
      meta:
        id: 3
        value: middlestr
    - processor: varGet
      meta:
        id: 4
        name: val
    - val3
  filter:
    - val1
    - val33

IfThenElse

This allows the use of logic in the for of “if… then… else…”.

The comparison is between two inputs, using a selected operator. The result of the Processor is then calculated using the meta in the “then” or “else” inputs.

** Note: The meta in then or else inputs is completely separate from the input to Lhs and RHS. This has been deliberate to ensure coverage of all use cases, even though there is an increased processing time. When “fragments” are created, they will speed this up significantly.

  • Inputs
    • Lhs
      The left-hand side of the equation.
    • Operator
      The comparison operator. Valid values are:
      • ==
      • ===
      • !=
      • >
      • >=
      • <
      • <=
    • Rhs
      The right-hand side of the equation.
    • Then
      Meta for the “then” part of the logic.
    • Else
      Meta for the “else” part of the logic.

Example:

processor: ifThenElse
meta:
  id: 1
  lhs: 0
  operator: "<"
  rhs: 
    processor: varGet
    meta:
      id: 2
      name: val
  then: "less than"
  else: 
    processor: url
    meta:
      id: 3
      method: get
      source: "http:/foo.bar.com/less/than"
      reportError: true

Merge

Takes the data sets (Object processor or URL endpoint) and performs a merge operation on them of merge, negation or union.

  • Inputs
    • sources
      Take 2 or literals processors and are the data sets to perform the action upon.
    • mergeType
      The action to perform on the data. This can only be literals of union, intersect or difference. It can also be a processor that provides any of the three literals from above.
    • unique
      Filter out duplicate values. The default is false.

Example:

processor: merge
meta:
  id: 1
  sources:
    - processor: object
      meta:
        id: 2
        attributes:
          - processor: field
            meta:
              id: 3
              key: key1
              val: val1
          - processor: field
            meta:
              id: 4
              key: key2
              val: val2
          - processor: field
            meta:
              id: 5
              key: key3
              val: val3
    - processor: object
      meta:
        id: 6
        attributes:
          - processor: field
            meta:
              id: 7
              key: key1
              val: val1
          - processor: field
            meta:
              id: 8
              key: key4
              val: val4
          - processor: field
            meta:
              id: 9
              key: key5
              val: val5
  mergeType: union
  unique: true

Object

Create a custom object from inputs. This is useful for creating an output object from selected fields. You can use the Field processor for key/value pairs, or other processors or literals to create single values. If then is only a single input then Object will populate with only this (rather than each input in an array), so a normalised Url output can be placed in Output for further processing.

  • Inputs
    • attributes
      A single value or an array of values that belong to the object. Accepts processor or literal.

Example:

processor: object
meta:
  id: 1
  attributes:
    - processor: field
      meta:
        id: 2
        key: key1
        val: val1
    - processor: field
      meta:
        id: 3
        key: key4
        val: val4
    - processor: field
      meta:
        id: 4
        key: key5
        val: val5
    - val4
    - val5
    - val6

ProcessorsAll

Fetch data on all processors available to the application.

ResourceJson

CRUD for a resource in JSON format.

An input YAML’s syntax will be validated against required sections, and all processor inputs will be validated against the processor cardinality and input rules. If any validations fail, an error object with details of the error will be returned, and the resource will not be saved.

  • Inputs
    • Method
      Can be a string or a processor. The HTTP method for the resource (only used if fetching or deleting a resource).
    • Appid
      Can be a literal or a processor. The application ID for the resource (only used if fetching or deleting a resource).
    • Noun
      Can be a literal or a processor. The noun for the resource (only used if fetching or deleting a resource).
    • Verb
      Can be a literal or a processor. The verb for the resource (only used if fetching or deleting a resource).
    • Resource
      Can be a literal or a processor. The JSON string or file for the resource (only used if creating or updating a resource).

This can be in the form of a processor or file attached to the request.

ResourceYaml

CRUD for a resource in YAML format.

An input YAML’s syntax will be validated against required sections, and all processor inputs will be validated against the processor cardinality and input rules. If any validations fail, an error object with details of the error will be returned, and the resource will not be saved.

  • Inputs
    • Method
      Can be a string or a processor. The HTTP method for the resource (only used if fetching or deleting a resource).
    • Appid
      Can be a literal or a processor. The application ID for the resource (only used if fetching or deleting a resource).
    • Noun
      Can be a literal or a processor. The noun for the resource (only used if fetching or deleting a resource).
    • Verb
      Can be a literal or a processor. The verb for the resource (only used if fetching or deleting a resource).
    • Resource
      Can be a literal or a processor. The YAML string or file for the resource (only used if creating or updating a resource).

This can be in the form of a processor or file attached to the request.

ResourceSwagger

Input the YAML from Swagger and generate the basic Resources in the Database.

These Resources will be populated with:

  • name
  • description
  • uri noun & uri verb from the path
  • method

The security section will be created with user validation with a role of Consumer, using GET variable token. His can be edited later.

The processor section will be empty, and needs populating.

There will be new (unused) section called “parameters”. This gives the meta for all input parameters described in the Swagger YAML, i.e. URI fragments, GET/POST variables, contents of the request Body, etc. You can use the Meta from

  • Inputs
    • Resource
      This can be in the form of a processor or file attached to the request.

Sort

This allows the sorting of the values in an Object.

The sorting is ascending or descending, based on common keys within the Object’s Fields. If a key does not exist in a Field, then an error object will be returned.

  • Inputs
    • Object
      The object to perform the sort on.
    • Key
      The key within the Fields to be used in comparison.
    • Direction
      The direction to sort in, can be “asc” or “desc”.

Example:

processor: sort
meta:
  id: 1
  values:
    - val1
    - val2
    - val3
    - val4
    - val5
    - 35
    - 456
  direction: asc

VarBody

Contains the contents of the body from the request.

  • Inputs
    None.

Example:

processor: varBody
meta:

VarBool

A boolean variable. Throws an error if the input is not a Boolean. It accepts yes, no, true, false, 0, 1

  • Inputs
    • Value
      The value of the variable. Accepts processor or literal.

Example

processor: varBool
meta:
  id: 1
  value: yes

VarPersistent

A persistently stored variable, that extends beyond the life of a single request. This allows you to create, edit, fetch and delete a frequently used value at any time.

  • Inputs
    • Name
      The name of the variable. Accepts processor or literal.
    • Value
      The value of the variable. This is only used if saving the variable.
    • Operation
      The operation to be performed on the variable. Can be ‘save’, ‘delete’ or ‘fetch’ or a processor producing a string of the above.

Example

processor: varPersistent
meta:
  id: 1
  name: varName
  value: varVal
  operation: save

VarRand

Generate a random variable.

  • Inputs
    • Length
      The length of the variable. Accepts processor or integer.
    • Lower
      Include lower case alpha characters.
    • Upper
      Include upper-case alpha characters.
    • Numeric
      Include numeric characters.
    • special
      Include non-alphanumeric characters.

Example

processor: varRand
meta:
  id: 1
  length: 16
  lower: true
  upper: true
  numeric: true
  special: true

VarTemporary

A temporarily stored variable, that extends only during the life of a single request. This allows you to create, edit, fetch and delete the value at any time in the meta.

  • Inputs
    • Name
      The name of the variable. Accepts processor or literal.
    • Value
      The value of the variable. This is only used if saving the variable.
    • Operation
      The operation to be performed on the variable. Can be ‘save’, ‘delete’ or ‘fetch’ or a processor producing a string of the above.

Example

processor: varPersistent

  id: 1

VarUri

A number of the position of the value within the URI. This counts like an index, where 0 is the position after noun/verb:

0 in datagator.yourtemp.website/v1/<app_code>/<noun>/<verb>/foo = foo
1 in datagator.yourtemp.website/v1/<app_code>/<noun>/<verb>/foo/bar = bar
etc
  • Inputs
    • Value
      The index of the variable in the URI. Accepts processor or integer.

Example

processor: varUri
meta:
  id: 1
  index: 0
