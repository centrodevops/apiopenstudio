Accounts, Applications and resources

An Account is the highest grouping level, and it can contain multiple applications. Essentially, it can be an account for an entire company, department or project. This is the billing point.

An Application is a grouping of rest calls and an account can have many applications. It is a logical grouping of REST calls into a specific API for a mobile application, a project, website/s or some other consumer of data.

Resources are individual API REST calls that belong to an application.

Users and Roles

There are 4 tiers of public access to the API resources:

  1. Owner – the owner has permission granted by the system to monitor overall use of the account’s applications, can grant administrator access to any user (including themselves), is responsible for the billing.
  2. Administrator – the user has permission granted by the system to create/edit/delete applications, and can view resource and application statistics.
  3. Developer – the user has permission granted by the system create, edit and delete resources for an application.
  4. Consumer – the user has permission granted by the system to request a token, and use that token to access protected resources for an application.

A user can own many applications and there is no limit to how many roles a user can be granted or to how many accounts, applications and resources.

Further down the roadmap, there is a plan to allow applications to create custom roles, so that users can be easily grouped and easily granted access to specific resources.
