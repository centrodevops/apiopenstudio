validation

  • TokenConsumer
  • TokenDeveloper
  • TokenSysAdmin
  • TokenUser
  • TokenRole

These processors can only be used in the validation section of the resource.

Token

Validate the request, requiring the user to have a valid token.

  • Inputs
    • Token
      The consumers token. Can be a processor.

TokenConsumer

Validate the request, requiring the user to have a valid token and a role of consumer

  • Inputs
    • Token
      The consumers token. Can be a processor.

TokenDeveloper

Validate the request, requiring the user to have a valid token and a role of developer

  • Inputs
    • Token
      The consumers token. Can be a processor.

TokenSysAdmin

Validate the request, requiring the user to have a valid token and a role of system admin

  • Inputs
    • Token
      The consumers token. Can be a processor.

TokenUser

Validate the request, requiring the user to have a valid token and is in an array of usernames.

  • Inputs
    • Token
      The consumers token. Can be a processor.
    • Usernames
      1 or more usernames. Can be processors or literals

TokenRole

Validate the request, requiring the user to have a valid token and is having a defined role.

  • Inputs
    • Token
      The consumers token. Can be a processor.
    • Role
      The consumer’s role. Can be processors or literals
