In general, any resources that you create will belong to your application only, and no other user that is not associated with your application can access it unless you create a resource without any security or token validation (i.e. open to anyone).

However, there are some resources that are available to all, or requiring a specific role. These are listed below.

User Login

Login with your user account. Note, the password is not stored in the DB so if you lose the password, your account will have to be manually reset.

Does not require a token.

  • URL
    /api/<app_id>/user/login/?username=<username>&password=<password>
  • Method
    POST
  • Header
    Accept: application/*
  • Response
    Token or an error object.

YAML import

Import a YAML document into Datagator to create a new resource, or update an existing one. The resource will be automatically applied to the app ID in the URL. The user must have developer role for that application. The YAML can either be in an attached file or as a url-encoded GET variable named ‘yaml’.

Requires developer role.

  • URL
    /api/<app_id>/resource/yaml/?token=<token>
  • Method
    POST
  • Parameters
    Token (mandatory)
  • Header
    Accept: application/*
  • Response
    True/false or an error object.

YAML export

Fetch an existing YAML document from Datagator.

Requires developer role.

  • URL
    /api/<app_id>/resource/yaml/?token=<token>&method=<method>&noun=<noun>&verb=<verb>
  • Method
    POST
  • Parameters
    Token (mandatory) Method (mandatory) Noun (mandatory) Verb (mandatory)
  • Header
    Accept: application/text
  • Response
    True/false or an error object.

The user must have developer role for that application. The json can either be in an attached file or as a url-encoded GET variable named ‘json’.

Requires developer role.

  • URL

  • Method
    POST
  • Parameters
    Token (mandatory)
  • Header
    Accept: application/json,application/xml,*/*
  • Response
    True/false or an error object.

Swagger import

Create resources from a Swagger YAML document.

Requires developer role.

  • URL
    /api/<app_id>/resource/json/?token=<token>&method=<method>&noun=<noun>&verb=<verb>
  • Method
    POST
    *Parameters
    Token (mandatory) Method (mandatory) Noun (mandatory) Verb (mandatory)
  • Header
    Accept: application/json
  • Response
    True/false or an error object.

Resource delete

Delete an existing Resource from Datagator.

Requires developer role.

  • URL
    /api/<app_id>/resource/delete/?token=<token>&method=<method>&noun=<noun>&verb=<verb>
  • Method
    DELETE
  • Parameters
    Token (mandatory) Method (mandatory) Noun (mandatory) Verb (mandatory)
  • Header
    Accept: application/json,application/xml,*/*
  • Response
    True/false or an error object.

ProcessorsAll

Fetch a listing of all processors. This includes processor title, description, and a list of all possible inputs along with the input’s min/max number of values, allowed value types and description. .

Requires developer role.

  • URL
    /api/<app_id>/processor/all/?token=<token>
  • Method
    GET
  • Parameters
    Token (mandatory)
  • Header
    Accept: application/json,application/xml,*/*
  • Response
    A list of all processors available to the application or an error object.
