hello-world

The first thing we need to do is create the resource definition. We’ll use YAML for this, because it’s a very human readable format.

Open your favourite plain text editor and enter the following YAML text lines.

We will call this resource “Hello world”. The dictionary “name” is used to define this:

name: Hello world

Let’s also give the resource a description, which is longer and more descriptive that the title. This is defined ion the dictionary “description”:

description: Return a hello world string.

We want the URL for the resource to be api//hello/world. So we need to define the “uri” dictionary, which contains the mandatory “noun” and “verb” dictionaries:

uri:
   noun: hello
   verb: world

Because we are fetching data, the HTTP method will be GET. This is defined in the “method” dictionary:

method: get

We will want to cache the results for 5 minutes, so this is defined in the “ttl” dictionary, and the value is the seconds to cache: 

ttl: 360

We will not add any security to the resource. So this will be open to anyone to call (they will not need a token). So we can leave the security section empty.

Now we need to define the process section. This is where we define the logic, and here we will concatenate two strings: “Hello ” and “world!”.

The processor section must always begin with a processor, and the definition of a processor contains the “processor” (name of the processor) and “meta” (ID and inputs) dictionaries:

process:
   processor: <string>
   meta:
      id: n

The concatenate processor has a mandatory input of an array of values, called “sources”. Because this is the first processor, we’ll give this processor an ID of “1” (although we could use any string or integer for the ID). The input array “sources” will contain two string (“Hello ” and “world!”). So the “process” dictionary now becomes:

process:
   processor: concatenate
   meta:
      id: 1
      sources:
        - “Hello “
        - “world!”

The result will be in the response, in JSON format. This is the default, so we do not need an “output” section.

The final YAML that we have is:

name: Hello world

description: Return a hello world string.

uri:
   noun: hello
   verb: world

method: get

ttl: 360

process:
   processor: concatenate
   meta:
      id: 1
      sources:
        - “Hello “
        - “world!”

The empty lines between the dictionaries is not necessary, but have been added for readability. Do not use tabs, but use at least 2 spaces. Save the file as “hello_world.yaml”.

Now we need a developer token to upload the YAML. Using your developer account, make a user login request to Datagator with the following attributes:

  • URL
    http://datagator.yourtemp.website/api/3/user/login/
  • Method
    POST
  • Payload
    username=<username>
    password=<password>
  • Header
    Accept: application/json
    Content-Type: application/x-www-form-urlencoded

The response will be similar to:

{ "token": "45ceb6123bdfe20f7392a9747ff41a7c"}

Create a new Resource YAML post request to Datagator, with the following attributes:

  • URL
    http://datagator.yourtemp.website/api/3/resource/yaml/
  • Method
    POST
  • Payload
    token=<token>
  • Files
    • Name
      Resource
      *Value
      hello_world.yaml - that you previously saved.
  • Header
    Accept: application/json
    Content-Type: application/x-www-form-urlencoded

The response will be “true”, if successful or an error object if the request failed.

You can now make a GET request to your new resource:

http://datagator.yourtemp.website/api/3/hello/world

If everything has worked, the response should be:

Hello World!
