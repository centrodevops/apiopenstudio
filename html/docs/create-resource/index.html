create-resource

For a definition of YAML, and its syntax and standards, see http://www.yaml.org/spec/1.2/spec.html.

  1. login with your developer credentials, using the following POST REST call, with (Accept: application/json in the header is optional): http://datagator.yourtemp.website/api//user/login/?username=&password=
    1.1. To begin with, your logins will be new and there will be no password. So you must remember the password that you choose the first time that you login, because you will need it again to authenticate, when your current token expires and you need to fetch a new token.
  2. Copy the token you receive.
  3. Send your YAML to Datagator in either of the following ways:
    3.1. Save your YAML to a file, and make a POST REST call, with the file attached to: http://datagator.yourtemp.website/api//resource/yaml/?token= .
    3.2. Convert your YAML to a url-encoded string, and make a POST REST call to: http://datagator.yourtemp.website/api//resource/yaml/?token=&yaml= .
  4. If the result from the call is successful, test your call, using a valid consumer token if required.
