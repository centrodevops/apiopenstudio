merge-if-else

  • Basic template

  • Use fragments to remove repetition
  • Fetch the value from the URI

  • Validate for max range
  • Use fragments to remove repetition and complexity
  • Validate for min range
  • The final YAML

Here we have a complex tutorial. We want to merge the data from several sources, and then filter out one of them, based on a URI value. We also want to place validation on the URI value and place it within a range.

The range validation used here is not the most elegant approach, but it illustrates fragments and how things can be chained together.

Basic template

setup the basic template and fetch from an external source:

name: TutorialComplex

description: “A complex tutorial” 

uri:
  noun: tutorial
  verb: complex

method: get

ttl: 0

security:
  processor: tokenConsumer
  meta:
    id: 1
    token:
      processor: varGet
      meta:
        id: 2
        name: token

process: 
  processor: url
    meta:
    id: 3
    source: "https://randomuser.me/api/"

Merge multiple sources

Now extend the input into multiple inputs and merge them together, so we’ll edit the process dictionary:

process:    
  processor: merge
  meta:
    id: 4
    sources:
      - processor: url
        meta:
          id: 5
          source: "https://randomuser.me/api/"
      - processor: url
        meta:
          id: 6
          source: "https://randomuser.me/api/"
      - processor: url
        meta:
          id: 7
          source: "https://randomuser.me/api/"
      - processor: url
        meta:
          id: 8
          source: "https://randomuser.me/api/"
      - processor: url
        meta:
          id: 9
          source: "https://randomuser.me/api/"
      - processor: url
        meta:
          id: 10
          source: "https://randomuser.me/api/"
      - processor: url
        meta:
          id: 11
          source: "https://randomuser.me/api/"
      - processor: url
        meta:
          id: 12
          source: "https://randomuser.me/api/"
      - processor: url
        meta:
          id: 13
          source: "https://randomuser.me/api/"
      - processor: url
        meta:
          id: 14
          source: "https://randomuser.me/api/"
    mergeType: union
    unique: false

Use fragments to remove repetition

There’s a lot of repetition here, so let’s use a fragment:

process:    
  processor: merge
  meta:
    id: 4
    sources:
      - fragment: randomUser
      - fragment: randomUser
      - fragment: randomUser
      - fragment: randomUser
      - fragment: randomUser
      - fragment: randomUser
      - fragment: randomUser
      - fragment: randomUser
      - fragment: randomUser
      - fragment: randomUser
    mergeType: union
    unique: false

fragments:
  - fragment: randomUser
    meta:
      processor: url
        meta:
        id: 5
        source: "https://randomuser.me/api/"

Fetch the value from the URI

Now we need a processor to fetch the filter value from the URI (the first value after the <noun>/<verb>, i.e. /tutorial/complex/7):

processor: varUri
meta:
  id: 6
  index: 0

Filter a value from the list

Let’s take this value and use it to filter out an index. We’ll be doing some work on the URI value and it will get repeated, so let’s put it into a fragment now:

  processor: filter
  meta:
    id: 3
    values:    
      processor: merge
      meta:
        id: 4
        sources:
          - fragment: randomUser
          - fragment: randomUser
          - fragment: randomUser
          - fragment: randomUser
          - fragment: randomUser
          - fragment: randomUser
          - fragment: randomUser
          - fragment: randomUser
          - fragment: randomUser
          - fragment: randomUser
        mergeType: union
        unique: false
    filter:
      - fragment: filterNum

fragments:
  - fragment: randomUser
    meta:
      processor: url
        meta:
        id: 5
        source: "https://randomuser.me/api/"
  - fragment: filterNum
    meta:
      processor: varUri
      meta:
        id: 6
        index: 0

Validate for integer

Let’s validate that the value is an integer, so we wrap the result if VarUri with a VarInt:

- fragment: filterNum
  meta:
    processor: varInt
    meta:
      id: 7
      value:
        processor: varUri
        meta:
          id: 6
          index: 0

Validate for max range

We have only 10 inputs, what if the user entered 11? Let’s use wrap this in an IfThenElse processor:

- fragment: filterNum
  meta:
    processor: ifThenElse
    meta:
      id: 8
      lhs:
        processor: varInt
        meta:
          id: 7
          value:
            processor: varUri
            meta:
              id: 6
              index: 0
  operator: ">"
  rhs: 10
  then: 10
  else:
    processor: varInt
    meta:
      id: 9
      value:
        processor: varUri
        meta:
          id: 10
          index: 0

Use fragments to remove repetition and complexity

This is a bit messy, because we have repeated code again, so let’s make use of fragments again:

- fragment: uriVal
    processor: varInt
    meta:
      id: 9
      value:
        processor: varUri
        meta:
          id: 10
          index: 0

- fragment: filterNum
  meta:
    processor: ifThenElse
    meta:
      id: 8
      lhs:
        fragment: uriVal
      operator: ">"
      rhs: 10
      then: 10
      else: 
        fragment: uriVal

Validate for min range

Now add validation of less than 0:

- fragment: uriVal
    processor: varInt
    meta:
      id: 9
      value:
        processor: varUri
        meta:
          id: 10
          index: 0

- fragment: filterNum
  meta:
    processor: ifThenElse
    meta:
      id: 11
      lhs:
        processor: ifThenElse
        meta:
          id: 8
          lhs:
            fragment: uriVal
          operator: ">"
          rhs: 10
          then: 10
          else: 
            fragment: uriVal
      operator: "<"
      rhs: 0
      then: 0
      else: fragment: uriVal

The final YAML

name: TutorialComplex

description: “A complex tutorial” 

uri:
  noun: tutorial
  verb: complex

method: get

ttl: 0

security:
  processor: tokenConsumer
  meta:
    id: 1
    token:
      processor: varGet
      meta:
        id: 2
        name: token

processor: filter
meta:
  id: 3
  values:    
    processor: merge
    meta:
      id: 4
      sources:
        - fragment: randomUser
        - fragment: randomUser
        - fragment: randomUser
        - fragment: randomUser
        - fragment: randomUser
        - fragment: randomUser
        - fragment: randomUser
        - fragment: randomUser
        - fragment: randomUser
        - fragment: randomUser
      mergeType: union
      unique: false
  filter:
    - fragment: filterNum

fragments
  - fragment: randomUser
    meta:
      processor: url
        meta:
        id: 5
        source: "https://randomuser.me/api/"
  - fragment: uriVal
      processor: varInt
      meta:
        id: 6
        value:
          processor: varUri
          meta:
            id: 7
            index: 0

  - fragment: filterNum
    meta:
      processor: ifThenElse
      meta:
        id: 8
        lhs:
          processor: ifThenElse
          meta:
            id: 9
            lhs:
              fragment: uriVal
            operator: ">"
            rhs: 10
            then: 10
            else: 
              fragment: uriVal
        operator: "<"
        rhs: 0
        then: 0
        else: fragment: uriVal
