external-api-data

This tutorial will pull in data from an external API (https://randomuser.me/api/) and output it in various formats.

The process will be similar to the hello world tutorial. Open you favourite plain text editor and add the basics, this time we will have no caching:

name: External API
description: Fetch a result from an external API.
uri:
   noun: external
   verb: api
method: get
ttl: 360

The resource should have some security, because it’s using an external API, and we don’t want to get banned for spamming. So we’ll use the Consumer security processor. This requires a token from a user with Consumer role. The input of the token string will be the output from the VarGet processor – that is, it will get the GET variable it finds into the VarConsumer processor

security:
  processor: tokenConsumer
  meta:
    id: 1
    token:
      processor: varGet
      meta:
        id: 2
        name: token

This time, we will use the Url endpoint processor. This takes several inputs, and since this external endpoint does not require any authentication, we only need the mandatory “method”, “source”, “reportError” and “normalize” inputs:

process:
  processor: url
  meta:
    id: 3
    source: “https://randomuser.me/api/”
    method: get
    reportError: true
    normalise: true

** ReportError will stop processing if there is an error fetching the external data and return an error object - this is useful if the data is critical to the processing, during development or you need an application to react to bad data.

** Normalize will convert the input data from a pure string to an object. If the input string is not a JSON or XML string, then the object will take the format of {data: }

We can already set the response format with the header:

Accept: {application|text/{json|xml|text|html}

But perhaps we also want to receive an email with the data as well as in the response. So we need to create an output dictionary:

output:
  - response
  - email:
      to:
        - foo@bar.com
        - john@naala.com.au
      format: xml
