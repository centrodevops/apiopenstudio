overview

Datagator is an API aimed at developers of all skill levels and small to medium size digital agencies, who want to rapidly create API’s that contain logic but do not want to spend to time programming a new API or setting up a server.

Datagator introduces an extra tier to application design and architecture, called the “Data Logic Tier”. Data logic couples with the familiar Data Tier in n-tier or multi-tier architecture but is separate from business logic, database and other tiers. It only applies to selecting, processing and formatting data received from external sources into a format to be digested by applications or websites.

Datagator is more than an aggregator, and allows data from multiple sources to be processed and cached before it is delivered. This is particularly useful to save bandwidth and processing time of unnecessary data.

The only data stored by Datagator, at present are the login credentials of external systems.

The programmatic logic for processing data is broken down into basic elements and encapsulated in modules called “Processors”. Each of these processors are simple logic blocks, like foreach and if/else, operations like merge or concatenate, scientific and mathematical formula, variables (strict or loosely typed), objects and key/value pairs, different inputs (endpoints).

This greatly speeds up the description and creation of a logic flow to process data because there is no need to program the elements, and does not require any knowledge of a specific language or it’s syntax. The only understanding that is required is logic.
